Our AI Philosophy: Not every problem is a GenAI problem.

RetainIQ uses AI where it creates measurable value — and classical algorithms everywhere else. We don't use LLMs to rank offers because a gradient boosted tree does it in 2ms with full explainability. We don't use embeddings for eligibility because a graph traversal is deterministic and auditable. The right tool for the right job.

Machine Learning

Churn Prediction

LightGBM gradient boosted trees with 5 feature groups. Sub-millisecond inference, SHAP explanations, Bayesian cold-start adaptation per tenant.

Features: Usage(35%) + Billing(25%) + Contact(20%) + Lifecycle(12%) + Competitive(8%)
Machine Learning

Outcome Learning

Accept/decline feedback feeds weekly retraining. Model AUC improves measurably over 30-day windows. Automatic rollback if drift exceeds 5%.

Kafka → Feature Store → Argo Training → MLflow Registry → Canary Deploy
Classical Algorithm

Multi-Objective Ranking

Configurable weighted scoring — not a black box. Operators tune the weights to match their business priorities.

score = 0.45*retention + 0.30*margin − 0.15*spend_cap + 0.10*context
Classical Algorithm

Catalog Graph

Products are nodes with edges: incompatibility, upgrade-from, bundle-with. Graph traversal ensures offers are logically consistent — no ML needed.

Graph: 6 products × 3 edge types × per-market regulatory metadata
